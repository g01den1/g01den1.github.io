1、标准库概述：

1.1、Hello World：

​ 首先，先来看看最小的C++程序：

​ 这个程序只定义了一个main函数，但是这个函数没有任何参数，也没有任何需要执行的指令。

​ 每个程序中必须有一个名字为 main() 的函数，程序会从这个函数开始执行。

​ 下面是一个经典的输出 Hello World 的程序。

​ #include \<iostream\> 指示编译器去包含位于 iostream 里的标准 I/O 功能的声明。如果没有那些声明，表达式 std::cout << "Hello World !!\n";将会没有任何意义。

​ 运算符<<代表讲它的第二个参数写到第一个参数里。在这一具体情况中，字符串文字量 “Hello World!!\n” 将会被写进标准输出流 std::cout 。字符串文字量就是由双引号括起来的一个字符序列。在字符串文字量中，由反斜线字符 \ 后跟一个字符表示的是某个特殊字符。在这儿出现的 \n 是换行符，因此，串中的是字符 Hello World!!，后跟一个换行符。

1.2、标准库名字空间：

​ 标准库定义在一个成为 std 的名字空间里。这也就是为什么cout前要加个 std 的原因。

​ 标准库的每一种功能都是通过某个像 <iostream>这样的头文件提供的。

​ 为了简要起见，有的时候我们会在前面加上 using namespace std把这个名字作为全局的。一般来说，将所有名字都采用全局的是不好的。

1.3、输出：

​ 库 iostream 为每种内部类型定义了相应的输出方式。退一步说，为用户定义类型定义一种输出方式也是很容易的。默认情况下，送到 cout 的输出值都将被转换为字符的序列。

1.4、字符串：

​ 标准库提供了一个 string 类型，作为前面所用的字符串文字量的补充。这个 string 类型提供了许多很有用的字符串操作，例如串拼接等。

​ 对于字符串而言，+号就代表拼接。当然， 也可以使用+=来进行拼接。

​ 字符串也可以通过运算符进行比较，比如 == 、>、< 等等。

​ substr()运算返回一个字符串，它就是函数参数所指明的那个子串的副本。第一个参数是在本字符串的一个下标，第二个参数是所需子串的长度。

​ replace()操作用一个值替换掉指定的子串。
